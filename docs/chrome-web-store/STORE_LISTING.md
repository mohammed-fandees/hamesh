## Detailed description

### English

```
Hamesh gives the web a margin.

Books and manuscripts have margins — a quiet space beside the text where a
reader leaves a note, a question, a connection to something else. Hamesh
brings that back to the web.

HOW IT WORKS
1. Activate Hamesh from the toolbar icon or the Alt+H shortcut.
2. Click the exact element on the page you want to annotate — a paragraph,
   a heading, a button, a line of code.
3. Write a short note. It saves instantly.
4. A small marker stays next to that element. Click it anytime to reopen,
   edit, or delete the note.
5. Leave the page, come back later (even after reloading or restarting your
   browser) — your note is restored right where you left it.

WHO IT'S FOR
- Designers and developers annotating a component, a design detail, or a
  line in the documentation.
- Researchers marking exactly where an argument or a source lives.
- Knowledge workers keeping the "why" next to the "what," across everything
  they read.
- Students and lifelong learners leaving a trail of attention to return to.

PRIVACY
Hamesh stores every note locally on your device using your browser's own
extension storage. There is no account, no backend server, and no network
requests of any kind — nothing is transmitted, analyzed, or shared. See our
privacy policy for full details.

CURRENT SCOPE (v0.1.0)
This is a focused first release: plain-text notes, one device, no cloud
sync yet. It supports both English and Arabic interfaces with right-to-left
layout. Hamesh is not yet available for other browsers.

Hamesh is not a bookmarking tool, not a sticky-note app, and not a
generic writing assistant — it is a quiet layer for thinking in context,
exactly where that thought belongs.
```

Approximate length: ~1,480 characters (well within typical detailed-description tolerances; no official hard cap was found in current documentation — see `README.md` sourcing notes. If a cap is enforced live in the dashboard, trim the "WHO IT'S FOR" section first, as it's the least load-bearing for review purposes).

## Language / localization strategy

- **Primary listing language for initial submission:** English, because Chrome Web Store review and most discovery traffic defaults to the store's configured locale, and English maximizes reviewer clarity during the first submission.
- **Add Arabic as a second listing language** via the dashboard's localization feature once the initial English listing is approved and live — this lets Arabic-speaking users see the Arabic copy above automatically when their Chrome locale is Arabic, without maintaining two separate listings.
- **In-product language:** already implemented and verified — Hamesh's UI follows the browser's extension locale (`src/ui/i18n.ts`), defaulting to English and switching to Arabic (RTL) automatically; this is independent of the store listing language and works today regardless of which listing language is live.
- Keep brand naming consistent everywhere: **هامش** and **Hamesh**, never anglicized or re-transliterated differently across fields.
